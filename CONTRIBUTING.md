# Contributing to expert-review-panel

Thanks for helping improve expert-review-panel.

High-value contribution areas:
- new domain-specific expert libraries
- stronger anti-groupthink safeguards
- clearer output templates
- better eval coverage and validation tooling
- README / installation / release improvements

Suggested workflow:
1. Open an issue or discussion if the change affects workflow or scope
2. Keep pull requests focused on one improvement area
3. When changing review logic, include concrete examples or test cases
4. If you add a new reference library, keep the structure consistent with the existing ones

Good PRs are specific, scoped, and easy to verify.
