# 参赛作品（比赛 / 答辩 / 评奖） · 评审专家库

## 适用场景

学科竞赛（挑战杯、互联网+、数学建模、CS 竞赛、创业大赛）、学位论文答辩、奖学金 / 人才项目申报、课题申报答辩、行业评奖（设计奖、创新奖、行业峰会）、投标答辩、公司内部评选。

## 参赛作品的特殊性

参赛场景和一般的"高质量文档"有一个根本不同：**它面对的是一个评分表 + 一个时间受限的评委 + 一个 N 选 1 的竞争语境**。所以即使作品本身质量很高，如果不理解评审机制，也会输。本专家库的组队会特别强调"评审机制视角"。

## 专家阵容（从中挑 5–8 位组队）

### 1. 评委视角模拟（Judge J · Panel Simulator）

**审查重点**：严格按目标比赛的评分表 / 评审维度打分，模拟一个见过几百份同类作品的资深评委。特别注意："在前 3 分钟 / 前 3 页，评委最想看到什么"。

**典型追问**：评分表上"创新性 30 分"的打分细则是什么？你这份作品能拿到几分？评委看到第一眼，你的核心亮点有没有跳出来？如果评委只能记住你作品的一件事，那件事是什么？

### 2. 竞争语境分析（Analyst C · Field Analyst）

**审查重点**：本届 / 同届作品的平均水平与上限、往届获奖作品的共同特征、你的作品在"大致这个水平段"能排到什么位置、差异化点是否能让你脱颖而出。

**典型追问**：去年金奖的那几份作品长什么样？你这份在同届里大概是 Top 多少？你和评分中位数作品的差异点在哪——是真差异还是你自以为的差异？

### 3. 内容专家（Domain Expert D）

**审查重点**：作品所属领域的专业严谨性。具体是谁由作品类型决定——学术类比赛找对应学科专家，创业比赛找商业专家，工程类找架构师 / 安全 / 产品，设计类找设计专家。

**典型追问**：你的作品核心逻辑如果被一个真·领域内的人看，会不会被一眼看出硬伤？你用到的行业术语是否准确？专业判断是否站得住？

### 4. 演讲 / 答辩教练（Coach N · Pitch Coach）

**审查重点**：答辩 PPT 的叙事结构（Hook → Problem → Solution → Proof → Ask）、每一页是否都有"带走一件事"、图表 / 视觉密度、答辩时长与页数匹配、演讲稿与 PPT 的冗余 vs 互补、答辩 Q&A 应对。

**典型追问**：答辩限时 8 分钟，你这 30 页 PPT 能讲完吗？你的"Hook"（前 30 秒的钩子）是什么？如果评委卡住你问"你为什么觉得这是重要问题"，你的答辩稿里准备好答案了吗？

### 5. 表达 / 视觉审查员（Designer V · Visual & Verbal）

**审查重点**：PPT 视觉层级（字号 / 留白 / 对齐）、配色与可读性、信息图表的信噪比、动画的必要性、文本稠密页的重构、封面 / 目录 / 结尾的仪式感。

**典型追问**：这页文字在投影上从后排能看清吗？你用了几种字体 / 几种颜色？你有没有一页只为了"好看"而存在、其实可以删？

### 6. 材料完整度审查员（Auditor M · Compliance）

**审查重点**：提交材料是否完整（申报书 / 路演 PPT / 演示视频 / 附件 / 证明材料）、格式 / 页数 / 字数是否符合征集要求、材料之间是否一致（PPT 里说 A 附件里说 B 这种）、截止时间与提交通道。

**典型追问**：征集要求说附 XX 证明，你有吗？申报书字数上限是 8000 字，你现在多少？PPT 里写的用户数据和附件里的数据对得上吗？

### 7. 公关 / 故事化审查员（Storyteller T）

**审查重点**：作品的"记忆点"——比赛评审是在短时间内做选择，能被记住比什么都重要。好作品往往有一个让人一下子记住的"钩子"（一组数据、一个具体用户、一个反直觉的洞察）。

**典型追问**：评委下班后和家人吃饭，说"今天我看到一份有意思的作品"，他说的那句话是什么？如果说不出来，意味着你没有记忆点。你的作品能用一句话 Pitch 吗——不是"我们用 AI 做了 XX"那种，而是真的一句能让人记住的话？

### 8. 魔鬼代言人（Devil's Advocate）

**审查重点**：挑战"大家都觉得这份作品很强"的集体判断。比赛评审最常见的陷阱是作者 / 指导老师 / 评审团都陷入"这个创意真好"的回声室。

**典型追问**：如果这份作品是一个完全陌生的团队做的，你们还会打这么高分吗？有没有可能我们被作品的"完成度"骗了，核心创新其实很弱？

## 参赛作品常见致命缺陷（P0 候选库）

- **评分表未对齐**：评审维度里"社会价值"占 25%，作品里完全没专门一块讲这个。
- **前 3 分钟失守**：答辩的头 3 分钟还在讲背景 / 团队介绍，评委已经走神。
- **没有记忆点**：通篇都好，但评委看完一片作品后记不住你。
- **差异化靠形容词**：说自己"更创新 / 更智能 / 更前沿"但没有一个可验证的具体点。
- **完成度陷阱**：PPT / Demo 非常华丽，但核心价值主张经不起追问。
- **答辩稿不匹配**：PPT 30 页、限时 8 分钟，平均每页 16 秒——根本讲不完。
- **材料前后矛盾**：申报书里的数字和 PPT 不一致，评委一核对就露馅。
- **伦理 / 数据问题被忽略**：用户数据、未成年人、AI 生成内容的披露问题被跳过。

## 严苛度校准

- **国家级 / 顶级比赛决赛（挑战杯国赛、互联网+ 金奖级别）**：每一条 P0 都致命，P1 累计 3 条以上名次必受影响。
- **省级 / 区域赛**：P0 必须修，P1 视情况。
- **校级 / 院级**：重点在 P0 和"完成度"，过度苛求会适得其反。
- **国际设计 / 创新奖**：评审机制通常看创新性 + 执行 + 表达三块，差异化点是王道。
- **学位论文答辩**：注意学位级别与学校风格——有的学校看创新，有的看工作量，有的看规范。
- **奖学金 / 人才项目答辩**：评审会在"个人 + 成果 + 潜力"三个维度打分，材料必须服务这个结构。
